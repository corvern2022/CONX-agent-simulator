# CONX Foundation AI Agent 시뮬레이터 — 개선 설계 문서

**날짜:** 2026-04-11  
**목적:** 내부 팀 프레젠테이션용 리디자인  
**파일:** `index.html` (단일 파일 유지)

---

## 배경 및 목표

현재 시뮬레이터는 텍스트 입력 → 키워드 매칭 → 단계별 애니메이션 구조다. 기능은 동작하지만 내부 팀 프레젠테이션 용도로는 세 가지 문제가 있다:

1. **시각 임팩트 부족** — 발표 자료로서 인상이 약함
2. **전체 구조 파악 불가** — 8개 시나리오와 3단계 라우팅 체계를 한눈에 볼 방법이 없음
3. **입력 UX** — 발표 중 타이핑은 실수 위험이 있고 흐름을 끊음

**개선 목표:** "AI가 이런 상황을 이렇게 처리한다"를 라이브 데모로 보여주는 발표 자료. 클릭 한 번으로 시나리오를 선택하고, 전체 시스템 구조도 한눈에 파악 가능해야 한다.

---

## 사용 방식

- 발표자가 프로젝터 앞에서 직접 조작하는 라이브 데모
- 슬라이드 대신 이 페이지 자체가 발표 자료
- 팀원들이 프로젝터 화면을 보면서 따라감

---

## 전체 구조

단일 `index.html`, 세 개의 수직 섹션으로 구성. 섹션 간 스무스 스크롤.

```
┌─────────────────────────────┐
│  섹션 1: 시스템 오버뷰      │  ← 발표 시작 시 전체 구조 소개
├─────────────────────────────┤
│  섹션 2: 시나리오 선택      │  ← 카드 클릭 or 텍스트 입력
├─────────────────────────────┤
│  섹션 3: 플로우 애니메이션  │  ← 선택 후 자동 스크롤
└─────────────────────────────┘
```

---

## 섹션 1 — 시스템 오버뷰

### 목적
팀이 "어떤 상황이 어떤 경로로 처리되는가"를 한눈에 이해하는 맥락 제공.

### 레이아웃
- 페이지 상단, 제목 + 부제 + 3열 라우팅 맵
- 3열: Green / Yellow / Red 각각의 승인 라우팅 흐름을 세로 칩 체인으로 표시

### 각 열 구성
```
[Green / Yellow / Red 레벨 뱃지]
[레벨 설명: "일반 운영" / "외부 커뮤니케이션" / "고위험 의사결정"]
[해당 시나리오 예시 태그들]

[역할 칩 1]
     ↓
[역할 칩 2]
     ↓
[역할 칩 3]
    ...
```

### 역할 칩 내용

**Green (3단계):**  
실무 에이전트 → CoS → 배포

**Yellow (5단계):**  
실무 에이전트 → Risk & Compliance → CoS → CEO Agent → 배포

**Red (7단계):**  
실무 에이전트 → Risk & Compliance → CoS → CEO Agent → 내부 협의체 (Human) → CEO Agent → 배포

### 디자인 포인트
- 열 배경색: 각 레벨 연한 색조 (green-tint / yellow-tint / red-tint)
- 테두리: 해당 레벨 강조색
- Human 개입 칩은 시각적으로 구분 (아이콘 또는 다른 스타일)

---

## 섹션 2 — 시나리오 선택

### 목적
발표자가 실수 없이 빠르게 시나리오를 선택. 타이핑 불필요.

### 레이아웃
리스크 레벨별 그룹 헤딩 + 카드 그리드 + 커스텀 입력창

```
🔴 HIGH RISK
[카드] [카드] [카드]

🟡 MEDIUM RISK
[카드] [카드]

🟢 LOW RISK
[카드] [카드] [카드]

──────────────────────────────
💬 등록되지 않은 상황 직접 입력:
[입력창                    ] [시작]
──────────────────────────────
```

### 카드 구성 (각 시나리오 카드)
- 리스크 레벨 색상 좌측 액센트 바
- 시나리오 제목 (크게)
- 담당 에이전트명
- 단계 수 뱃지 (예: "7단계")

### 상호작용
- 카드 클릭 → 선택 상태 강조 (테두리, 배경 강조) → 섹션 3으로 자동 스크롤 + 시뮬레이션 시작
- 텍스트 입력 후 버튼 클릭 or Enter → 기존 키워드 매칭 로직 유지 → 섹션 3으로 스크롤

### 카드 목록 (8개)

| ID | 제목 | 에이전트 | 레벨 | 단계 |
|----|------|---------|------|------|
| disclosure | 공시 등록 | 공시 Agent | Yellow | 5 |
| exchange | 거래소 커뮤니케이션 | BD Agent | Yellow | 5 |
| mou | 파트너사 MOU 체결 | BD + 법무/재무 | Red | 7 |
| community | 일반 커뮤니티 공지 | 커뮤니티 마케팅 Agent | Green | 3 |
| proposal | 온체인 프로포절 제출 | 거버넌스 | Red | 7 |
| legal | 법무 계약 검토 | 법무/재무 | Red | 7 |
| tech | 테크 문서 업데이트 | 테크 Agent | Green | 3 |
| cs | 유저 CS 대응 | CS Agent | Green | 3 |

## 섹션 3 — 플로우 애니메이션

### 목적
선택된 시나리오의 처리 흐름을 단계별로 극적으로 보여줌. 프레젠테이션의 핵심 무대.

### 헤더 바
```
[리스크 레벨 뱃지] [시나리오 제목]        [담당 에이전트]
[프로그레스 바: ████████████░░░░░░]  3 / 7 단계
```

### 스텝 카드

**완료된 스텝:**
- 흐린 배경, 체크(✅) 아이콘, 역할명 + 결과만 표시

**현재 진행 중 스텝:**
- 강조 배경 + 테두리, ⚡ 아이콘
- 역할명 (크게)
- 설명 텍스트 타이핑 애니메이션
- 결과 텍스트 페이드인

**미완료 스텝:**
- 숨겨진 상태, 차례가 되면 슬라이드 업으로 등장

### 커넥터
스텝 간 세로 선 커넥터. 완료 시 레벨 색상으로 변함.

### 완료 후
- 마지막 스텝 완료 시 완료 메시지 표시
- "다른 시나리오 보기" 버튼 → 섹션 2로 스크롤

### 기존 대비 변경점
- 스텝 카드 크기 확대 (프로젝터 가독성)
- 프로그레스 바 추가
- 완료/진행중/대기 상태 시각적 구분 강화
- 섹션 2와 연동 (카드 선택 → 자동 스크롤)

## 전체 디자인 원칙

### 타이포그래피
- 기존 Inter 폰트 유지
- 베이스 폰트 사이즈 소폭 확대 (프로젝터 가독성)
- 스텝 역할명: `1.2rem` 이상, 굵게

### 색상
- 기존 따뜻한 베이지 계열 유지 (`#f5f4ef`, `#2b2a28`)
- Green: `#1d9e75`, Yellow: `#ba7517`, Red: `#a32d2d` 유지
- 레벨별 연한 배경색 활용 강화

### 반응형
- 기존 모바일 대응 유지
- 오버뷰 섹션: 모바일에서 3열 → 1열 스택

### 기술 스택
- 단일 `index.html` 유지
- 외부 의존성 없음
- 기존 시나리오 데이터, 키워드 매칭 로직, 애니메이션 코드 재활용
